# Introduction à Doklear

## Introduction
Doklear est une plateforme d’audit documentaire et d’assistance RAG connectée à votre documentation existante.
Utilisez Doklear pour identifier rapidement les risques documentaires : incohérences, duplication, structure faible, et obsolescence.

Doklear n’impose pas une “méthode parfaite” : connectez vos contenus tels qu’ils existent (Markdown, HTML, export Confluence, dépôts Git, fichiers).
Ensuite, lancez un audit et priorisez les corrections.

## Prérequis
- Créez un compte Doklear.
- Créez un workspace (un workspace = un client / un produit / un périmètre documentaire).
- Préparez au moins une source de documentation (GitHub, URLs, fichiers).

## Instructions
1. Connectez-vous à Doklear.
2. Cliquez sur **Workspaces**.
3. Cliquez sur **Créer un workspace**.
4. Donnez un nom clair (ex: `doklear-demo`).
5. Cliquez sur **Valider**.
6. Dirigez-vous vers l’onglet **Ingestion**.
7. Ajoutez une source (GitHub ou URL) pour commencer.

## Conclusion
Une fois la documentation ingérée, lancez un audit pour obtenir un état des lieux.
Ensuite, utilisez le chat pour valider rapidement une procédure et vérifier si la doc couvre bien une question réelle.
